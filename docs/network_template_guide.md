## 0. 一句话省力填法（先看这一段）

> **必须人写**：姓名、可信度（0-5）、AI 标准化特征、认识。
> **可以留空**：所属行业、公司、职务、城市、潜在需求、合作价值、资源类型 ——
> 这些**导入完之后跑一次 `lodestar enrich --owner X --apply`**，AI 会从你写的
> 「AI 标准化特征」和「备注」自由文本里把它们结构化抽出来。

> **「关系类型」列已被删除**。"我有没有联系到这个人"完全由「可信度」这一列承载：
> **填 0 = 未联系（系统不建 Me 边、靠他人引荐到达）；填 1-5 = 已联系（按强度建边）**。
> 一列一个数字，不会再有"关系类型 + 可信度"互相冲突的填法。

也就是说：**精力别花在分列填字段上，花在写好「AI 标准化特征」这一栏短标签 +
把「认识」列里跨公司的人写全**。AI 解析的隐私模型见 §六。

## 二、5 条铁律（看完再动笔）

1. **不要改列名、不要换列顺序、不要改 sheet 名**。系统按列名匹配，错一个字就丢一整列数据。
2. **姓名是唯一主键**：同名会被自动合并；如果你有两个同名不同人（比如两个"张伟"），请用 `张伟（中金）` `张伟（红杉）` 这种括号写法人为消歧。
3. **公司列填好就省掉一半的"认识"列**：同一个公司里的人会被系统自动连成同事（强度 4）。所以同事关系**不需要**手写到「认识」里。
4. **「认识」列是这套表能不能用的关键**：每个人写他认识圈里的 3-5 个**已经在表里**的人就够了，不用穷举。写到表外的人会被警告并跳过。
5. **没数据就留空**。**不要**填 `-` `无` `NA` `N/A` `暂无`——这些会被原样存成字符串，污染搜索。

## 三、13 列怎么填

> v3（2026-04-20 下午）把「关系类型」列删掉了——「是否联系到」完全可以从「可信度」推断（0=未联系，1-5=已联系），多一列反而制造二义性。

| 列名 | 必填 | 怎么填 | 反例 |
|---|---|---|---|
| 序号 | 否 | 自己排顺序，留空也行 | — |
| **姓名** | ★ | 真名；同名时加括号消歧 | `张总`（太模糊） |
| 所属行业 | 推荐 | 大类标签，多值用 `;` 分隔 | `投资银行;TMT` |
| 公司 | 推荐 | 当前任职公司全称；前后任用 `;` 分隔 | `中金公司;原高盛` |
| 职务 | 推荐 | 当前职位 | `衍生品 MD` |
| 城市 | 选填 | 常驻城市 | `上海;香港` |
| **AI 标准化特征** | 推荐 | **短句关键词，分号分隔**——这是语义检索吃的字段 | 见下方专门一节 |
| **可信度（言行一致性0-5分）** | ★ | **整张表的核心列，单列承载两件事**：`0` = 未联系（不建 Me 边）；`1-5` = 已联系，数字 = 边的强度。**留空 = 默认 3**，所以未联系的人**务必显式填 0** | 不要填 `5+` 或 `中` |
| 合作价值（0-5） | 选填 | 0-5 整数。和「可信度」**正交**：可信度=人靠不靠谱、合作价值=商业潜力大不大。**朋友圈类联系人没必要评估时直接留空** | 别和可信度填一样 |
| 潜在需求 | 推荐 | 此人在找什么；用于反向撮合 | `家办客户;海外上市资源` |
| 资源类型 | 选填 | 此人能**提供**什么资源，多选：`资金;项目;服务;技术;人脉`。和「潜在需求」是供需两端 | 别填行业、别填具体项目名 |
| **认识** | ★ | 见下方专门一节 | — |
| 备注 | 选填 | 自由文本，**不进搜索打分**，只在详情页展示 | — |

> 「合作价值」和「资源类型」是 v2 新加的两列，主要服务于 Tommy 这种**机构合作**语境的填表人。Richard 这种**朋友圈**语境没必要每行都填——评估了就填、没评估就留空，不会报错。

## 四、三个最容易写错的字段

### 1) AI 标准化特征 —— 写"标签串"，不要写整段话

这是系统做语义匹配的字段，请把对此人的关键画像浓缩成 **5-10 个短标签**，用 `;` 分隔。越具体越好。

```
✅ 港大金融;衍生品;交际广;外派经验;善于做局
✅ 清华五道口;稳健派;保险机构视角;做固收
❌ 这个人挺厉害的，我跟他认识很多年了，他在中金做衍生品。   （整段话，命中率会低）
```

### 2) 「认识」—— 这是系统造"引荐路径"的唯一原料

格式：

```
人名1(强度,描述); 人名2(强度); 人名3(描述); 人名4
```

- 多个人之间用 `;` 或换行分隔。
- 括号里 **数字 = 1-5 强度**，**文字 = 关系描述**，**两个都可省**。
- 顺序无所谓：`(4,大学同学)` 和 `(大学同学,4)` 都对。
- 写一个方向就行：你给 A 写了 `认识 B`，系统会自动建 A-B 双向边，**不需要**再去 B 那一行写"认识 A"。
- 写到表外的人会被跳过（导入时有警告）——**所以引用的名字要先在表里有自己的一行**。

```
✅ 刘思敏(5,中金同事); 沈南鹏(2,中金时见过几次); 张磊
✅ 周景明(3); 邱国鹭(高毅路演)
❌ 认识很多人。   （没法解析）
❌ 刘思敏，沈南鹏。 （括号写法都没有也可以，但失去了强度信息）
```

> **省力建议**：同一个公司的同事**不要**写到「认识」列里——已经在「公司」字段填过的人会被自动连成同事。「认识」只用来描述**跨公司**的人脉，比如校友、前同事、合作过的对家、饭局认识的等。

### 3) 「可信度」—— 单列承载一切（v3 设计正交化）

整张表里**唯一**用来表达「我和这个人的关系」的列，只填一个 0-5 的整数：

| 值 | 含义 | 系统行为 |
|---|---|---|
| **0** | 未联系：我还没直接接触过此人 | **不建** Me 边；命中搜索时自动算 1-3 跳引荐路径 |
| **1** | 点头之交（旧版「弱认识」就是这一档） | 建 Me 边，强度 = 1 |
| **2** | 认识，偶尔联系 | 建 Me 边，强度 = 2 |
| **3** | 普通朋友 | 建 Me 边，强度 = 3 |
| **4** | 熟人，常合作 | 建 Me 边，强度 = 4 |
| **5** | 核心铁磁，深度信任 | 建 Me 边，强度 = 5 |

> ⚠️ **留空 = 默认 3**，会被当成普通朋友建直边。**未联系的人务必显式填 0**，否则引荐路径就推不出来了。

> ⚠️ **不要在表里预先标谁是「想接触的目标」**。
>
> "我下次想找谁、想做什么"是你在 web 端搜索框里说的那句自然语言，由 LLM 解析意图后从全表（含**已联系 + 未联系**）一起匹配并算路径——任何人都可能是某次查询的最佳匹配。**事实**（联系/未联系）∥ **强度**（1-5）∥ **意图**（搜索那句话）三个维度严格分开，没有任何字段同时表达多个维度。

> **历史包袱**：v2 模板里曾经有过「关系类型」列（值为 `直接` / `弱认识` / `已联系` / `未联系` / `目标`）和单独的「可信度」列。今天 (v3) 把「关系类型」彻底删了，因为：
>
> - `弱认识` 本质就是 `可信度=1` 的快捷方式，重复表达；
> - `已联系 + 可信度=0` 是合法但矛盾的填法（系统按"已联系"建边、强度却是 0），多列 = 多歧义；
> - `目标` 这个词混淆了"事实"（这人我还没联系到）和"意图"（这人我想找）。
>
> v3 单列模型让填表心智降到最低：每个人就一个 0-5 的整数。
> 老 xlsx 里残留的「关系类型」列会被 importer 静默忽略，不影响导入。

## 五、可选的「关系」sheet —— 给信息特别丰富的关系用

主表里「认识」列已经够用。但如果某条关系**信息特别多**（怎么认识、什么频率、上次什么时候联系过的细节），写到「关系」sheet 更合适。5 列：

| 甲 | 乙 | 强度 | 关系 | 频率 |
|---|---|---|---|---|
| 陈维国 | 沈南鹏 | 3 | 中金时多次合作（项目顾问） | yearly |

- `频率` 只能填：`weekly` / `monthly` / `quarterly` / `yearly` / `rare`。
- 这里的内容会**覆盖**主表「认识」列里同一对的内容。

## 六、AI 自动解析：能帮你省掉哪些列、不能替代什么

从 2026-04-20 起，Lodestar 接入了一个"读 free text、补结构化字段"的 AI 解析步骤。**没有它你照样能用**——所有列你都可以全部手填；但**用了它能让填表工作量直接砍一半**。

### 6.1 它能帮你做什么（"机器读得懂的列"）

读你写的 **「AI 标准化特征」+「备注」+「职务」** 这三个 free text 字段，自动**追加补全**以下 4 类结构化属性：

| 字段 | 来源举例 |
|---|---|
| **公司列** `companies[]` | "港大金融;**中金**;衍生品" → 提取 `中金公司` |
| **城市列** `cities[]` | "**长居香港**，紫光最大个人 LP" → 提取 `香港` |
| **职务标签** `titles[]` | "FOF **基金经理**;稳健派" → 提取 `基金经理` |
| **语义标签** `tags[]` | "私募 FOF;港股 IPO;长期共赢" → 标准化成检索 tags |

> "追加而非覆盖" —— 你已经填好的 `公司`、`城市` 列**永远不会被改写**，AI 只补"你没填的"。所以填表时不必担心被 AI 弄丢手填内容。

补完之后还会跑一步 **同事网推断**（`infer-colleagues`）：把"两个人在同一家公司"这件事真正落成同事边。**你不用手写"A 认识 B 都在中金"——AI 抽出公司，系统就自动把同公司的人连成同事网**。Tommy 的表跑下来增加了 62 条同事边，没多填一个字。

### 6.2 它**不能**帮你做什么（"机器猜不出来的列"）

| 列 | 为什么必须手填 |
|---|---|
| **姓名** | 主键，AI 不会替你录入新人 |
| **可信度（含 0=未联系）** | 这是你对此人言行一致性的私人判断 + 你和他有没有联系到的事实，AI 没有这种主观信号 |
| **认识列** | **核心断言**：跨公司的人脉关系（校友、前同事、共同朋友、饭局认识）只有你知道。AI 即使读 bio 也不可能猜出"你和某某一起喝过酒"这种事。**这是构成多跳引荐路径的唯一原料，必须人写**。 |

### 6.3 隐私模型 —— 名字和公司是怎么发到云端的

我们用阿里云百炼的 LLM API。但**你的真实姓名和公司名永远不会以明文发出去**：

```
本地原文                    →   发给云端的内容
─────────────────────────────────────────────
"陈维国"                   →   "P001"
"沈南鹏"                   →   "P037"
"中金公司"（已经在表里）    →   "C002"
"红杉资本中国"（已经在表里） →   "C014"
"国信证券深圳分公司"        →   "C002深圳分公司"  ← 长名优先匹配，不串
"九方智投"（第一次出现）    →   "九方智投"        ← 还没结构化的新公司只能明文给
                                                   AI 让它抽，下次跑就有 token 了
```

LLM 的回复也是 `Pxxx` / `Cxxx` 格式，本地反向映射回真名后再入库。**云端只能看到 token 化后的版本，看不到任何真实姓名 + 已经登记过的公司名**。

> 一个收敛过程：第一次跑时新公司是明文，落库后第二次跑就变成 `Cxxx` 了。长跑下来云端能看到的公司数会渐近于 0。

### 6.4 怎么跑

#### 命令行（按顺序跑，每一步都幂等）

```bash
# 第 1 步：导入 Excel
uv run lodestar import examples/template.xlsx --owner you

# 第 2 步：让 AI 补 公司/城市/职务/标签
#   --only-missing 默认只补空字段；想强制重跑用 --all
uv run lodestar enrich --owner you --dry-run    # 先看
uv run lodestar enrich --owner you --apply      # 再写

# 第 3 步：把同一家公司的不同写法合并
#   （比如「国泰君安」「国泰海通」「国君」其实是合并后的同一家）
uv run lodestar normalize-companies --owner you --dry-run
uv run lodestar normalize-companies --owner you --apply

# 第 4 步：把 AI 抽到的公司物化成同事网
uv run lodestar infer-colleagues --owner you --apply
```

> **第 3 步是新加的**（2026-04-20）。它有三档别名来源：
> 1. **`--builtin`** 默认开启 —— 内置一份高确定性中国金融机构合并改名表
>    （国泰海通 / 申万宏源 / 中金 ...）
> 2. **`--alias-file your_aliases.json`** —— 你可以维护一份自己的 alias 文件，
>    格式：`{"canonical": ["alias1", "alias2"]}`
> 3. **`--use-llm`** —— 让 LLM 把剩余公司名做二次聚类。**LLM 输出必须人工
>    review** 才能 apply（默认就是 `--dry-run`），因为 LLM 偶尔会把名字相近
>    但实际不同的公司错合（例如把"德恒律所" / "恒德" 误当成同一家）。

#### 前端 UI（不想用命令行的话）

| 入口 | 在哪里 | 干什么 |
|---|---|---|
| 新增联系人对话框 | 「AI 解析」按钮 | 你写完 bio/备注后点一下，AI 把可抽的字段填进对话框，你可改可弃 |
| 联系人详情页 | 「AI 重新解析」按钮 | 对当前这个人重跑一次解析，结果直接更新 |
| 顶栏「批量 AI 解析」 | 模态框 | 给当前 owner 起一个后台批量任务，带进度条 |

### 6.5 想最大化 AI 收益怎么填？

**关键就一句话：**「AI 标准化特征」这一栏写得越饱满，AI 越能帮你补**公司 / 城市 / 职务 / tags / 同事网**。其它列你可以全留空。

✅ **这种 bio 让 AI 能抽出 4 个字段 + 加入同事网**：
```
港大金融;中金衍生品 MD;长居香港;善于做局;客户基础是欧美对冲基金
```

❌ **这种只能抽 1 个字段，且不会进同事网**：
```
人挺好;经常一起喝酒
```

> 当然，如果你愿意手填公司/城市/职务，效果一样（甚至更准）—— AI 是为了"帮你省手填"，不是为了取代你已经做好的工作。

## 七、填完之后怎么验证

把文件交给负责导入的人，他会跑四步（每一步都幂等，可重跑）：

```bash
# (1) 导入 Excel
uv run lodestar import examples/richard_network.xlsx --owner richard

# (2) AI 补 公司/城市/职务/tags（详见 §六）
uv run lodestar enrich --owner richard --apply

# (3) 把同一家公司的不同写法合并（国泰君安 / 国泰海通 / 国君...）
uv run lodestar normalize-companies --owner richard --apply

# (4) 把 AI 抽到的公司物化成同事网
uv run lodestar infer-colleagues --owner richard --apply
```

每一步都会打印关键报告，注意看：

- **导入阶段** `[认识] 'A' → 'B' 未在表内找到，已跳过`
  ——「认识」里写到的人 B 在表里没有自己的一行。要么把 B 加进去，要么删这条引用。
- **enrich 阶段** 末尾的统计表会告诉你"X/Y 行新增了 N 个 companies / cities / tags / titles"。
  覆盖率太低（比如 `companies` 覆盖 < 30%）就说明你「AI 标准化特征」写得太薄，建议回头加厚。
- **normalize 阶段** 会列出"将合并的别名组"，例如
  `国泰海通证券 ← 国泰君安(3), 海通证券(4), 国君(2) ... 合计 13 人`。
  默认只用内置金融机构 alias 表；想让 LLM 二次清洗加 `--use-llm`，但 **LLM 输出
  必须 dry-run review 后再 `--apply`**，因为 LLM 偶尔会把名字相近、实际不同的
  公司错合。
- **infer-colleagues 阶段** 会列出"前 N 个公司簇"，比如 `中泰证券 4 人 → 6 条同事边`。
  如果还看到本应该是同一家的两家公司被分开成两簇，回到 §六.4 加进 `--alias-file` 再重跑。

没有报错 = 全部进库 + AI 已补完 + 同事网已建好。打开 `http://127.0.0.1:8765/` 切到你的 owner tab 就能看效果。

## 八、常见问题

**Q: 我有 200 多个人，必须全部填完才能导入吗？**
不用。可以先填高优先级的 30-50 个、跑一次导入看效果，再增量补充。重复跑导入是安全的。

**Q: 「AI 标准化特征」我能不能照搬职务和公司？**
不要。职务和公司已经有自己的列，重复了等于浪费。这一列要写**职务/公司之外**的画像（性格、专长、圈子定位、人脉特征等）。

**Q: 我还没联系到的人，可信度填什么？**
填 0。系统看到 0 就跳过 Me→TA 的边构建，确保只能靠他人「认识」列里的引荐路径抵达。**留空 ≠ 0**——留空会被默认成 3。

**Q: 我和某人只是点头之交、刚加了微信，怎么填？**
可信度填 1。这就是以前所谓的「弱认识」档，现在统一用 1 来表达。

**Q: 一个人有多份工作怎么办？**
「公司」列用 `;` 分隔列出，例如 `中金公司;原高盛`。每家公司的同事都会自动和此人连边。

**Q: 我和某人是远房亲戚也算"认识"吗？**
算。可信度按你对此人的实际熟悉程度填（1-5），「认识」列里写"远房亲戚"就行。

**Q: 我可以只填姓名 + 可信度 + 认识列、剩下全交给 AI 吗？**
可以。但「AI 标准化特征」**至少**写 3-5 个标签 —— 否则 AI 没东西可读，没法补出公司/城市/职务。其它列全留空没问题。

**Q: AI 解析会不会把我的姓名/公司发到外网？**
不会发明文。所有人名换成 `Pxxx` token，所有"已经在表里登记过"的公司换成 `Cxxx` token。只有"还没结构化的新公司"以明文给 AI 让它抽取（这是任务本身的目的）—— 一旦 AI 把它入库，下次跑就也是 token 了。详见 §6.3。

**Q: 我手填了「公司」列，AI 会不会把我填的覆盖掉？**
不会。AI 解析永远只**追加**到现有列表，绝不覆盖。同理「城市」「tags」也是。

**Q: 跑完 AI 解析之后，我看到一个公司被抽错了怎么办？**
直接在前端联系人详情面板手动改即可。下次跑 enrich 时 AI 会基于改后的版本继续抽，不会再回退。

## 九、给 Tommy 的额外提示：原 16 列 `tommy_network.xlsx` 怎么映射

如果你手上是早期的 `tommy_network.xlsx`（曾叫 `contacts.xlsx`，16 列
的机构合作画像表），按下表整理到模板里：

| Tommy 原列 | 模板归宿 |
|---|---|
| 姓名 | → 姓名 |
| 行业 | → 所属行业 |
| 身份职位 | → 职务 |
| 地域 | → 城市 |
| 潜在需求 | → 潜在需求 |
| 可信度（承诺一致性 0-5分） | → 可信度（言行一致性0-5分） |
| **合作价值评分（0-5）** | → **合作价值（0-5）** |
| **资源类型（资金/项目/服务/技术/人脉）** | → **资源类型** |
| 核心标签（机构自营/家办/...） | → 所属行业（拼到一起，例 `保险资管;家办`） |
| 主要背景 | → 备注 |
| 可合作业务范围 | → AI 标准化特征（浓缩成短标签） |
| 单笔可投资金额 | → 备注（口径不一，不进搜索） |
| 风险承受能力（激进/平衡/稳健） | → AI 标准化特征（如 `平衡` `稳健派`） |
| 共赢性（长/中/短） | → AI 标准化特征（如 `长期共赢` `短期诉求`） |
| 兴趣偏好 | → AI 标准化特征 |
| 关系阶段（A/B/C/D） | → **直接映射成可信度（0-5）**：`A`→4-5（熟、信得过）；`B`→3（普通朋友）；`C`→1（点头之交） 或 0（确实还没接触上、想被引荐）；`D`→1-2 + 备注写"曾合作但结果差" |

> 折叠 8 列进 AI 标准化特征 / 备注 看起来"有损"，但这些字段在搜索时被字符串化检索是够用的。如果未来确实需要按"风险承受=稳健"做精确筛选，再升级为独立列也只需要 30 分钟改 importer，xlsx 不用动。
