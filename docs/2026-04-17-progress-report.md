## 一句话讲产品

> **输入"我想做什么事"，系统在你的人脉库里找到最合适的人、给出最短的联系路径、并说明为什么是他。**

类似 GitHub Copilot 之于代码，Lodestar 之于"人脉这个私有数据库"。

## 今日交付

### 1. 项目从零跑起来，可在浏览器里实际操作
- Python 环境、SQLite（含向量扩展 `sqlite-vec`）、LLM / Embedding 接入（阿里云百炼）全部打通。
- 一条命令即可拉起本地服务：`uv run lodestar serve`，默认端口 `8765`。
- 前端是一个单页应用：力导向图谱 + 搜索框 + 详情面板 + 统计面板。

### 2. UI 重做成行业级专业视觉
- 参考 Linear / Warp / Sentry 的设计语言，做成深色 obsidian 风格。
- 节点按行业上色、按可信度发光；Me 节点居中高亮；命中节点和路径用主色点亮。
- 去掉了原型里的所有 emoji，换成自绘 SVG 图标；撮合功能不再用"爱心"，改为更中性的"引荐"。

### 3. Excel 导入全面升级，解决"只有我连别人、别人不互连"的问题
旧版数据只记录 `我 ↔ 某人` 这一维，导致图谱是一个巨大的星形，搜不出"最短人脉路径"——因为路径永远是 `我 → 候选人`，没有中间人。

今日的升级：
- Excel 模板新增 **`认识`** 列：可以写 `建国哥(4,大学同学); 李总(3,前同事); 陈博(老朋友)`，直接建人-人关系边。
- 新增 **`公司`** / **`城市`** 列：同一家公司的人自动被视为同事（关系强度 4）。
- 可选的 **`关系`** sheet：关系上下文丰富的边（频率、描述）单独维护，覆盖"认识"列。
- 重复导入幂等：同名自动合并、同对关系自动去重，同事改完 Excel 随时重跑。

### 5. 搜索准确度重大修复：从"找同类"变成"找能帮我的人"
这是今天最难的一个问题。早期测试"想募 AI 算力种子轮"，系统把**其他 AI 创业者**排在前面——因为他们的 bio 跟这句话语义最像。

根因是四层漏掉的逻辑，全部修掉：

| 层级 | 旧行为 | 新行为 |
| :-- | :-- | :-- |
| LLM 意图解析 | 提取 goal 里的角色（创始人/CEO） | 提取"能帮我的那个人的画像"（投资人/FOF/BD） |
| 关键词匹配 | 连 `需求` 列也加分 | 不再匹配"需求"——那是对方缺什么，不是对方能给什么 |
| 混合检索 | Vector / keyword / topic 三路等权 | 三路加权：helper 角色/行业 占重、话题关键词只做弱补充 |
| 路径打分 | 关系强度 5 碾压 relevance 0.05 差 | 关系强度只做 ≤10% tiebreak，相关性重回主导 |

**修复前后对比** —— 查询"想募 AI 算力种子轮"：

| # | 修复前 | 修复后 |
| :-- | :-- | :-- |
| 1 | 苏晓明（AI 算力创业者）❌ | **吴翰林（FOF 管理合伙人）** ✅ |
| 2 | 罗子昂（芯片创业）❌ | 周景明（量化 CIO / 管理合伙人）✅ |
| 3 | 蔡嘉怡（AI 创业）❌ | 徐文（FOF 研究员）✅ |

其他验证过的 query：
- "想跟杭州金融局聊地方补贴政策" → **#1 黄启明（杭州金融工作局处长）**
- "想让量化策略通过合规审查" → **#1 李昊然（风控总监，券商合规出身）**

### 6. 终极形态：多跳引荐（晚上追加）

前 5 项完成后发现还有一个**方向性偏差**——用户原意是"找到某一个或几个人，中间通过若干个中间人，把整条路径高亮出来"，但当时系统默认所有人都是直接好友，所以路径只有 1 跳。

补齐了四个改动，让「从我到**不认识的「未联系」人物**」的多跳路径成为一等公民：

**① Excel 新增 `关系类型` 列**
- `直接`（默认）= 可信度作为 Me 边强度
- `弱认识` = 只建强度 1 的 Me 边（点头之交）
- `未联系` = **不建 Me 边**——此人只能通过别人的 `认识` 列被抵达

**② Demo 数据补进 7 个名人型「未联系」人物**
沈南鹏 / 张磊 / 朱啸虎 / 雷军 / 张一鸣 / 段永平 / 邱国鹭。每人 3-4 条来自直接好友的引荐线：
```
陈维国 → 沈南鹏（中金时多次合作）
蔡嘉怡 → 朱啸虎（金沙江领投倾听 AI 天使轮）
罗子昂 → 雷军（墨子芯给小米澎湃做 IP 授权）
白总   → 段永平（温州老乡饭局共同朋友）
...
```

**③ 搜索 API 分桶 + PathFinder 打标签**
- 每条结果带 `path_kind`：`direct` / `weak` / `target`
- 返回拆成三个独立数组，各自 top-k，避免直接好友把「未联系」人物挤出结果
- `indirect` 类多跳路径用更软的 `hops^0.5` 惩罚（`direct` 仍是 `hops^1.3`），让 2 跳「未联系」不被 1 跳直接压死
- 不可达的「未联系」人物直接丢弃，不再返回路径强度 0 的噪声

**④ 前端改为"路径优先 UI"**
- 结果面板拆三段：🟡 需要引荐的人 / 🟢 直接认识的人 / ⚪ 弱认识
- 路径以 `我 → 中间人 → 候选人` 的胶囊链表示，末端节点金色高亮
- 点击任一行 → 图上**只**高亮这一条路径并自动居中
- 搜索完成后自动聚焦 top-1 候选，第一眼就看到杀手级用例

**端到端验证（3 个跨 cluster query）**：

| 查询 | 命中（"需引荐"桶 / 旧名 target 桶） | 路径 | 直接桶 top1 |
| :-- | :-- | :-- | :-- |
| 想募 AI 算力种子轮 | **朱啸虎** | 我 → 蔡嘉怡 → 朱啸虎 | 苏晓明 |
| 找消费品成熟投资人聊价值投资 | **段永平** | 我 → 白总 → 段永平 | 何文博 |
| 想接触字节系战投谈 AI 应用 | **张一鸣** | 我 → 蔡嘉怡 → 张一鸣 | 乔楠（战投） |

数据盘：36 人（含 7 个「未联系」）、124 条边、29 条 Me 边、8 条同公司自动补边。

## 最终列格式模板

输出物：`examples/template.xlsx` —— 在原 `pyq.xlsx` 的 8 列基础上完成、可直接发同事填。

### 12 列定义（顺序即 Excel 列顺序）

| # | 列名 | 必填 | 说明 | 示例 |
| :-: | :-- | :-: | :-- | :-- |
| 1 | `序号` | - | 自增编号，留空也行 | `1` |
| 2 | **`姓名`** | ★ | 唯一主键；同名会合并 | `陈维国` |
| 3 | `所属行业` | ○ | 大类标签，多值 `;` 分隔 | `投资银行` |
| 4 | **`公司`** ⭐新 | ○ | 同公司两人自动成为同事（强度 4） | `中金公司` |
| 5 | `职务` | ○ | 当前职位，用于还原 bio | `衍生品 MD` |
| 6 | **`城市`** ⭐新 | - | 常驻城市，多城 `;` 分隔 | `香港` |
| 7 | `AI标准化特征` | ○ | 短评式画像；参与向量化匹配 | `港大金融;衍生品;交际广` |
| 8 | **`可信度`** `(0-5分)` | ★ | Me 边强度的基础值；「未联系」人物填 0 | `5` |
| 9 | `潜在需求` | ○ | 此人在找什么，用于反向撮合 | `家办客户;海外上市资源` |
| 10 | **`认识`** ⭐新 | ★ | **最关键的列**：本人认识圈里哪些人，建 peer 边 | `刘思敏(5,同事); 沈南鹏(2,中金见过)` |
| 11 | `备注` | - | 自由文本，不参与打分 | `人脉通天` |
| 12 | **`关系类型`** ⭐⭐今日新增 | ○ | 三选一 `直接` / `弱认识` / `未联系` | `直接` |

⭐新 = 本次迭代在 pyq.xlsx 基础上新增的列；⭐⭐ = 今天最后一版新增。
所有新列在导入器里都是可选的，旧 `pyq.xlsx` 不改动也能继续导入（老数据就是清一色『直接』）。

### 『认识』列语法

```
王毅; 建国哥(4,大学同学); 李总(3,前同事); 陈博(老朋友)
```
- 用 `;` 分隔每一项
- 括号内数字 = 关系强度 1-5（可省，默认 3）
- 括号内文字 = 关系描述（可省）
- 数字和描述顺序不限、用逗号分开
- 只需写一个方向，导入器自动建双向边
- 填的人若不在表里，导入时会警告并跳过

### 『关系类型』三档的业务含义

| 值 | 会建 Me 边吗？ | 用在什么人身上 |
| :-- | :-- | :-- |
| `直接`（或留空） | 会，强度 = 可信度 | 正经朋友、同事、老客户——大部分人 |
| `弱认识` | 会，但强度固定为 1 | 点头之交、饭局加微信、被介绍认识但没深聊过 |
| `未联系` | **不会建** | 你想接触、还没直接联系到的人。需要别人通过『认识』列引荐 |

这就是多跳引荐的结构化开关：**只要把想接触的潜在客户、想拜访的大佬、潜在投资人标成『未联系』**，系统就会自动找 1-3 跳的引荐路径并高亮。

### 可选的 `关系` sheet

如果某对关系信息特别丰富（怎么认识、频率），可单独在 `关系` sheet 写一行：

| 甲 | 乙 | 强度 | 关系 | 频率 |
| :-- | :-- | :-: | :-- | :-- |
| 陈维国 | 沈南鹏 | 3 | 中金时多次合作（项目顾问） | yearly |

频率取值：`weekly` / `monthly` / `quarterly` / `yearly` / `rare`（空默认 yearly）。Sheet 里的内容**覆盖**主表『认识』列里的同一对。

### 填写工作流

1. `cp examples/template.xlsx ~/我的人脉.xlsx`（或直接把模板发给同事）
2. 第 1 遍：只填**姓名 / 公司 / 职务 / 可信度**，把人录进来就行
3. 第 2 遍：回头补**认识**列——把"A 认识圈里的 B、C、D"写上。不用全写，写最关键的 3-5 个人就足够让图谱通起来
4. 第 3 遍：把"想接触但还没直接联系的人"也写进来，把**关系类型**改成『未联系』
5. `uv run lodestar import ~/我的人脉.xlsx` —— 重复导入是幂等的，改完再跑就行

## 下一步建议（按价值排序）

1. **接真实数据**：`examples/template.xlsx` 是干净的 12 列模板，直接发同事填。建议第一批 100+ 条。
2. **引荐话术生成**：系统找出路径后，调 LLM 生成一段给中间人的"微信转发文案"——让"找对人"到"真打招呼"只差一次复制粘贴。
3. **联系动作追踪**：记录"这条路径实际走通/没走通"的反馈，用来反向训练打分。
4. **邮箱 / IM 连接器**：从聊天记录和邮件里**自动提取**联系人和关系更新，不用一直手填 Excel。
5. **多人版本**：目前每个人一份本地数据。如果要团队共用，需要做 auth + 数据隔离。

## 技术债 / 已知限制

- 搜索结果的第 4 位往后，偶尔会混入"同公司同事但不相关"的人——这是**同公司自动建边**的副作用，真实数据下 bio 写得准会自然缓解。
- 所有关系打分目前是规则权重，未来可考虑用用户的点击/采用行为做 learn-to-rank。
- 目前多跳路径最多 3 跳（`max_hops=3`）。真实数据若稀疏，可能需要 4 跳才能抵达部分「未联系」人物；届时调 `LODESTAR_MAX_HOPS=4` 即可。
- 前端 Cytoscape 在 500+ 节点时会有卡顿，届时需上 WebGL renderer。
