# Lodestar 功能说明

> 一句话：输入「我想做一件什么事」，系统自动在你的人脉库里找出最合适的人，并把可行的引荐路径直接画在图上。

---

## TL;DR: 已实现的功能

可以直接在网页上完成下面这些事：

- 用一句中文描述需求，系统返回 Top 候选人 + 推荐理由
- 自动画出「我 → 中间人 → 候选人」的多跳引荐路径
- 把搜索命中节点和路径高亮在图谱上，其余压暗
- 两段呈现：**已联系（按强度排序）/ 需要引荐（未联系，附多跳路径）**
- 任选两个人，自动算出他们之间的可行路径
- 自动扫描全库，给出「你可以撮合 A 和 B」的双向引荐建议
- 点击节点查看完整资料：行业、公司、技能、需求、备注、相关人
- 按行业、可信度、关键词过滤当前图谱
- 在线新增联系人，可选立即生成向量
- 数据概览面板：联系人总数、关系总数、行业/公司/城市分布
- 帮助卡片 + 全套快捷键（搜索、新增、聚焦、退出等）

底层已实现：

- 本地 SQLite 数据库（含向量扩展），所有数据落地不外发
- LLM 需求解析 + 关键词匹配 + 向量检索三路融合
- 关系图实时构建，毫秒级路径推理
- Excel / CSV 批量导入，幂等更新，重复行/重复关系自动合并
- 二态关系语义建模 — **完全由「可信度」单列驱动**：`可信度=0` → 未联系（不建 Me 边、靠他人引荐抵达）；`可信度=1-5` → 已联系（按数字决定边强度）
- 同公司联系人自动建立同事关系
- 命令行与 Web 两套入口

## Web 界面功能详解

界面整体是一张全屏的人脉图谱，外加可折叠的工具栏与侧边面板。下面按界面区域逐项介绍。

### 顶部工具栏

- 自然语言搜索框：输入「我想做什么」回车，触发完整检索
- `RAW` 开关：跳过 LLM 解析，仅做关键词匹配（无 API key 也可用）
- 一键清空按钮：退出搜索态、清除高亮
- `双人路径` 按钮：进入两人路径模式
- `引荐` 按钮：弹出全库的双向撮合建议
- `新联系人` 按钮：打开新增联系人表单
- `?` 按钮：弹出快捷键和交互说明

### 主图谱画布

- 节点颜色 = 所属行业，节点大小 = 与“我”的关系强度
- “我”作为中心节点突出显示
- 默认低对比度的“环境模式”，避免视觉噪声
- 鼠标悬停节点：扩散显示其邻居
- 点击节点：打开右侧详情抽屉
- 滚轮缩放、拖拽平移
- 搜索完成后自动聚焦到 Top-1 候选节点

### 引荐路径面板（搜索结果）

- 顶部显示意图概要 + 两类结果数量（已联系 / 未联系需引荐）
- **你已联系的人**：可立即联系；按可信度（1-5）从高到低排序，1 即点头之交
- **需要引荐的人**：标记为「未联系」、靠多跳路径抵达；附 `我 → 中间人 → TA` 链
- 每条结果显示：排名、姓名、跳数、路径链、关系上下文、综合得分；「未联系」会带 ★ 未联系 chip
- 点击任意结果：图谱上仅高亮该条路径，并自动居中
- 路径以胶囊节点链表示「我 → 中间人 → 候选人」

### 过滤面板

- 行业筛选（chip 多选）
- 最低可信度滑块（可只看强关系）
- 名字 / 标签实时关键词过滤
- 一键重置；面板可关闭并保留浮动入口

### 节点详情抽屉

点击任一节点弹出，展示该联系人完整信息：

- 头像、姓名、行业
- 可信度（5 段进度条）
- 简介、标签、技能、公司、城市、需求、私人备注
- 相关人列表，点击可跳转到对方
- 操作：把此人设为路径起点 / 删除该联系人

### 聚焦摘要卡片

鼠标聚焦节点但未点开详情时，显示一个轻量摘要卡片：

- 头像、姓名、行业
- 与“我”的关系强度
- 邻居数量
- 「查看资料」入口直达完整详情

### 双人路径模式

- 顶部出现状态栏，提示依次点击起点与终点
- 计算两人之间的可行多跳路径
- 可随时退出回到默认视图

### 引荐推荐窗口

- 基于「需求 ↔ 标签 / 技能 / 公司」自动匹配
- 列出可由当前用户撮合的 A ⇌ B 双方
- 给出每条推荐的撮合理由
- 点击条目可查看双方关系

### 新增联系人表单

- 必填：姓名
- 可填：简介、标签、技能、公司、城市、潜在需求、私人备注
- 关系字段：与“我”的关系强度、联系频率、关系上下文
- 可选立即生成 embedding（需配置 API key）

### 数据概览面板

- 联系人总数、关系总数
- 行业 / 公司 / 城市等维度分布
- 浮动入口常驻右下角，便于随时打开

### 帮助与快捷键

- `/` 聚焦搜索框
- `N` 新增联系人
- `F` 全屏 / 退出
- `Esc` 清除高亮 / 关闭面板
- `?` 显示帮助
- 帮助卡片同时复述图谱常用交互（悬停扩散、点击详情、缩放平移、双人路径、引荐扫描）

### 通知提示

- 右下角轻量 toast 提示，反馈导入成功、保存成功、检索失败等操作结果

## 底层能力

界面上看到的一切，背后由以下能力支撑：

- **自然语言需求检索**：LLM 解析需求 → 关键词 / 向量 / 主题三路加权融合 → 得到候选人（任何人都可能命中，不是只有「未联系」才能成为结果）
- **可插拔 Stage-2 重排（可选）**：在 hybrid 召回与排序之间插入一层 reranker，用更精的判分把"角色对口"的人提到前面，治理 bi-encoder 的"同行业不同角色"断崖。三档：`none`（默认零开销）/ `llm`（多调一次 LLM）/ `bge`（本地 cross-encoder `bge-reranker-v2-m3`）。**推荐**装好 `uv sync --extra rerank` 后用 `bge`：聚合质量小幅领先 `llm`（R@5 0.766 vs 0.734、cliff-avoid 0.85 vs 0.80），延迟 ~12s/q vs ~17s/q、零 token 费。`llm` 在「找具体角色 + 强语义判断」类单 query 上仍最准，可临时切换。具体对照见 `docs/eval_2026-04-21.md`。
- **多跳路径推理**：实时构建 NetworkX 图，按关系强度做最短路径
- **二态关系语义建模**：完全由「可信度」单列驱动。`可信度=0` → 未联系（不建 Me 边，依赖他人引荐才能抵达）；`可信度=1-5` → 已联系（按数字决定边强度）。"是否联系到" 是事实，不要和 "想找谁" 这种意图混。
- **同公司自动补边**：同公司联系人自动视为同事，关系强度 4
- **去重与合并**：同名联系人自动合并；同对关系自动去重；导入幂等
- **向量检索**：基于 SQLite + sqlite-vec，本地存储，离线可用
- **多模型兼容**：LLM 与 embedding 走 OpenAI 兼容协议，支持百炼 / OpenAI / DeepSeek / Zhipu / Kimi 等

## Excel 模板与示例数据

仓库提供两个文件：

- `examples/template.xlsx`：干净空白模板，可直接发同事填
- `examples/demo_network.xlsx`：36 个虚构联系人 + 28 条关系边，已按下文标准填好

下文示例均来自 `examples/demo_network.xlsx`。

### 一、整体表结构

模板包含 3 个 sheet：

| Sheet | 作用 | 必填 |
| :-- | :-- | :-: |
| `联系人` | 主表，每行一个人 | ★ |
| `关系` | 单独维护信息丰富的人-人关系 | 可选 |
| `说明` | 填写说明，仅供同事阅读 | - |

`联系人` sheet 的 12 列：

| # | 列名 | 必填 | 说明 | 示例 |
| :-: | :-- | :-: | :-- | :-- |
| 1 | `序号` | - | 自增编号，留空也行 | `1` |
| 2 | `姓名` | ★ | 唯一主键，同名会合并 | `周景明` |
| 3 | `所属行业` | ○ | 大类标签，多值 `;` 分隔 | `私募基金` |
| 4 | `公司` | ○ | 同公司两人会自动建同事关系 | `沧海资本` |
| 5 | `职务` | ○ | 当前职位 | `CIO / 管理合伙人` |
| 6 | `城市` | - | 常驻城市，多城 `;` 分隔 | `上海` |
| 7 | `AI标准化特征` | ○ | 短评式画像，参与向量检索 | `清华数学系;高频因子;管理严格` |
| 8 | `可信度（言行一致性0-5分）` | ★ | **整张表的核心列，单列承载两件事**：`0` = 未联系（不建 Me 边）、`1-5` = 已联系，数字 = 边强度（1=点头之交，5=核心铁磁）。**留空默认 3**，未联系的人务必显式填 0 | `5` 或 `0` |
| 9 | `潜在需求` | ○ | 此人在找什么，用于反向撮合 | `LP 资金;机构投资者资源` |
| 10 | `认识` | ★ | 此人圈内认识哪些人，建人-人关系 | `郑雅婷(5,同事); 陈维国(3,业内)` |
| 11 | `备注` | - | 自由文本，不参与打分 | `圈内口碑极好` |

多值字段支持 `;` `，` `、` 分隔；空值留空，不要写 `-` `无` `NA`。

### 二、关键概念：`可信度` —— 单列承载一切

整张表里**唯一**用来表达「我和这个人的关系」的列。0-5 整数，单列同时编码两个维度：

| 值 | 是否建 "我 → 此人" 边 | 含义 |
| :-: | :-- | :-- |
| `0` | **不建** | 未联系：我还没直接接触到这个人，靠他人「认识」列里的引荐到达 |
| `1` | 建，强度 = 1 | 点头之交（旧版「弱认识」就是这一档） |
| `2` | 建，强度 = 2 | 认识，偶尔联系 |
| `3` | 建，强度 = 3 | 普通朋友 (留空也默认成这个) |
| `4` | 建，强度 = 4 | 熟人，常合作 |
| `5` | 建，强度 = 5 | 核心铁磁 |

> 当某次查询命中的人是 `可信度=0` 的人时，系统自动找 1–3 跳引荐路径放在搜索结果的 **「需要引荐的人」** 段落，并在结果卡片上挂一个 `★ 未联系` 的 chip。
>
> **设计理念（v3 正交化）**：以前曾经有过单独的 `关系类型` 列（`已联系`/`未联系`/`弱认识`），导致 `关系类型=已联系 + 可信度=0` 是合法但矛盾的填法、`弱认识` 又是 `可信度=1` 的冗余表达。今天 (v3) 把 `关系类型` 列彻底删除，**事实**（联系/未联系）∥ **强度**（1-5）∥ **意图**（搜索那句话）三个维度严格分开，没有任何字段同时表达多个维度。
>
> ⚠️ **不要在表里预先标谁是「想接触的目标」**。"下次想接触谁"是 web 端搜索框里那句自然语言决定的，由 LLM 解析意图后从全表（含已联系 + 未联系）一起匹配并算路径——任何人都可能是某次查询的最佳匹配。
>
> ⚠️ 老 Excel 里残留的「关系类型」列会被 importer 静默忽略，行为完全由「可信度」驱动。重导前不需要手工清理这一列。

### 三、关键概念：`认识` 列语法

```text
郑雅婷(5,同事); 孙伟(5,同事); 钟启勋(4,清华师兄); 陈维国(3,业内); 邱国鹭(2,高毅路演)
```

- 用 `;` 分隔每一项
- 括号内数字 = 关系强度 1–5（可省，默认 3）
- 括号内文字 = 关系描述（可省）
- 数字与描述顺序不限，用逗号分开
- 只需写一个方向，导入器自动建双向边
- 被引用的人若不在表里，导入时会警告并跳过

### 五、可选的 `关系` sheet

某对关系信息特别丰富时（怎么认识、频率），可单独在 `关系` sheet 写一行；该 sheet 内容会**覆盖**主表 `认识` 列里的同一对。

| 甲 | 乙 | 强度 | 关系 | 频率 |
| :-- | :-- | :-: | :-- | :-- |
| 周景明 | 陈维国 | 3 | 衍生品业务对接 | quarterly |
| 王博宇 | 吴翰林 | 3 | 宏观季度闭门会 | quarterly |
| 何文博 | 简宁 | 5 | FA 合作紧密 | weekly |
| 蔡嘉怡 | 乔楠 | 4 | 字节前同事 | monthly |
| 方亦辰 | 周景明 | 3 | 基金合规法律顾问 | quarterly |

`频率` 取值：`weekly` / `monthly` / `quarterly` / `yearly` / `rare`，留空默认 `yearly`。

### 六、推荐填写流程

1. 复制模板：`cp examples/template.xlsx ~/我的人脉.xlsx`，或把模板发给同事
2. **第 1 遍**：只填 `姓名` / `公司` / `职务` / `可信度`，先把人录进来
3. **第 2 遍**：补 `认识` 列，每个人写最关键的 3–5 条关系即可
4. **第 3 遍**：把"想接触但还没直接联系的人"也写进表，并把 `可信度` 填 `0`（系统看到 0 就不建 Me 边）
5. 导入：`uv run lodestar import ~/我的人脉.xlsx`，幂等更新

`examples/demo_network.xlsx` 含 9 个真实感强的圈层（量化私募 / 投行 / FOF / 政府国资 / AI 创业 / 学术 / 桥梁人物 / 医疗 / 销售渠道）和 7 个「未联系」人物（沈南鹏 / 张磊 / 朱啸虎 / 雷军 / 张一鸣 / 段永平 / 邱国鹭），可直接对照参考。

## 典型使用流程

### 场景一：导入数据后直接检索

1. 初始化数据库
2. 导入 Excel 或 CSV
3. 输入自然语言需求
4. 查看推荐联系人与引荐路径
5. 在图谱中确认路径是否合理

### 场景二：维护「未联系」类型人物

1. 在 Excel 中录入想接触但还没直接联系的人
2. 将其 `可信度` 填 `0`（这一个数字就足以告诉系统「未联系」，不再需要单独的 `关系类型` 列）
3. 补充中间人的 `认识` 关系
4. 重新导入数据
5. 用自然语言查询触发命中，看「需要引荐的人」段落里的多跳路径

### 场景三：演示或汇报

1. 导入 `examples/demo_network.xlsx`
2. 启动 Web 服务
3. 用若干自然语言需求做检索演示
4. 展示返回的命中人物、推荐理由和路径高亮

## 运行方式

### 命令行

```bash
uv run lodestar init --name "Your Name"
uv run lodestar import examples/demo_network.xlsx
uv run lodestar find "想募 AI 算力种子轮" --top 5
```

### Web 界面

```bash
uv run lodestar serve --host 0.0.0.0 --port 8765
```

默认访问地址：

- 本机访问：`http://127.0.0.1:8765/`
- 局域网 / 公网访问：`http://<服务器IP>:8765/`

## 当前适用范围

适合：

- 个人人脉管理
- 小规模私有联系人库
- 对内演示或试运行
- 基于 Excel 的初期数据建设

暂未覆盖：
