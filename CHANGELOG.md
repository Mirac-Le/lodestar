# Changelog

面向使用者的可见变更摘要；实现细节见 Git 历史。

**约定**：新条目写在顶部日期块内，按 **修复 / 变更 / 新增 / 文档** 分组（无则省略）。每条尽量 2～4 句，写清用户能感知的效果。

## 2026-04-22

### 修复

- 关系一句话解析里，模型常在 `context`、`rationale` 中照抄脱敏后的 `Pxxx` / `Cxxx`。现在在生成提案前做反向替换，UI 与持久化字段只出现真实人名与公司名。
- 未知 token（模型幻觉出的编号）保留原样，便于肉眼发现异常而非静默丢字。
- 新增单测覆盖「含 token 的 rationale/context → 反脱敏后无 P/C 编号」。

### 变更（Web）

- 关系抽屉：来源筛选芯片与列表徽标统一为「色点 + 文案」；手动 / 同事推断可点选过滤。
- 「AI 推断」芯片标为 L2 TODO 且禁用点击，与当前仅从一句话解析写入的 `ai_inferred` 能力范围区分开。
- 覆盖已有边时的提示按来源类型上色，悬停说明使用可读标签。

### 文档

- 新增 `docs/product-overview-2026-04-22.md`：按模块描述已实现能力与边界；脱敏链路用 ASCII 流程图与约束要点说明，避免占位图。
